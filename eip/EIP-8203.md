## Abstract

[ERC-8203](./eip-8203.md) defines a composable on-chain reputation system for autonomous agents. Each agent accumulates a Kinetic Reputation Score (KRS) in the range `[0, 10 000]` based on verified task completions reported via [ERC-8202](./eip-8202.md) escrows, dispute outcomes, and stake-weighted history decay. ERC-8203 provides a read interface that other standards (e.g., the H3 Geospatial Registry in [ERC-8201](./eip-8201.md)) can query without coupling to Opacus-specific storage.

## Motivation

Reputation in agent marketplaces is typically off-chain and silo-specific, requiring consumers to trust a centralized index. On-chain escrow outcomes already encode objective evidence of agent reliability. ERC-8203 aggregates these signals — completed tasks, dispute losses, voluntary stake — into a single portable score that any EVM contract can read.

Key properties this standard establishes:

* **Portability** — Any registry can query an agent's KRS without coupling to a specific backend.
* **Decay** — Scores decay over time towards a neutral baseline, ensuring stale history does not dominate.
* **Dispute sensitivity** — A single contested dispute reduces the score by more than accumulating equivalent successful tasks would recover.
* **Stake weighting** — Agents that voluntarily lock collateral above the minimum receive a multiplier bonus.

## Specification

### Constants

| Name | Value | Description |
|---|---|---|
| `MAX_SCORE` | `10 000` | Maximum achievable KRS |
| `NEUTRAL_SCORE` | `5 000` | Baseline score at genesis |
| `DECAY_RATE_BPS` | `10` | Score decays toward neutral by 0.1 % per epoch |
| `EPOCH_SECONDS` | `86 400` | One epoch equals 24 hours |
| `DISPUTE_PENALTY` | `500` | KRS points deducted per lost dispute |
| `TASK_REWARD` | `50` | KRS points awarded per verified task completion |
| `STAKE_MULTIPLIER_BPS` | `200` | Bonus BPS per 1 ETH staked above minimum |

### Structs

```solidity
struct ReputationRecord {
    uint256 score;           // current KRS [0, 10 000]
    uint256 tasksCompleted;
    uint256 disputesLost;
    uint256 stakedAmount;    // voluntary stake in wei (above bonding minimum)
    uint64  lastUpdatedAt;
    bool    initialized;
}
```

### Interface

```solidity
interface IERC8203 {
    function getScore(address agent) external view returns (uint256 score);

    function recordTaskCompletion(address agent, bytes32 escrowId) external;

    function recordDisputeLoss(address agent, bytes32 escrowId) external;

    function stakeReputation(uint256 amount) external payable;

    function unstakeReputation(uint256 amount) external;

    function applyDecay(address agent) external;

    function getRecord(address agent) external view returns (ReputationRecord memory);
}
```

### Methods

#### `getScore`

```solidity
function getScore(address agent) external view returns (uint256 score);
```

MUST return the agent's current KRS after applying virtual decay since `lastUpdatedAt`. MUST NOT modify state. If the agent has no record, MUST return `NEUTRAL_SCORE`.

#### `recordTaskCompletion`

```solidity
function recordTaskCompletion(address agent, bytes32 escrowId) external;
```

- MUST only be callable by a whitelisted escrow contract (e.g., an [ERC-8202](./eip-8202.md) deployment registered via governance).
- MUST increment `tasksCompleted` by 1.
- MUST increase `score` by `TASK_REWARD`, capped at `MAX_SCORE`.
- MUST update `lastUpdatedAt = block.timestamp`.
- MUST emit `TaskCompleted(agent, escrowId, score)`.

#### `recordDisputeLoss`

```solidity
function recordDisputeLoss(address agent, bytes32 escrowId) external;
```

- MUST only be callable by a whitelisted escrow contract.
- MUST increment `disputesLost` by 1.
- MUST decrease `score` by `DISPUTE_PENALTY`, floored at `0`.
- MUST update `lastUpdatedAt = block.timestamp`.
- MUST emit `DisputeLost(agent, escrowId, score)`.

#### `stakeReputation`

```solidity
function stakeReputation(uint256 amount) external payable;
```

- MUST accept native ETH only (`msg.value == amount`).
- MUST add `amount` to `stakedAmount`.
- MUST recompute `score` boost: `boost = min((stakedAmount / 1e18) * STAKE_MULTIPLIER_BPS, 1000)`.
- MUST emit `ReputationStaked(agent, amount, score)`.

#### `unstakeReputation`

```solidity
function unstakeReputation(uint256 amount) external;
```

- `amount` MUST be ≤ `stakedAmount`.
- MUST reduce `stakedAmount` by `amount` and recompute boost accordingly.
- MUST transfer `amount` back to `msg.sender`.
- MUST emit `ReputationUnstaked(msg.sender, amount, score)`.

#### `applyDecay`

```solidity
function applyDecay(address agent) external;
```

- MUST compute elapsed epochs since `lastUpdatedAt`.
- Per epoch: `score = score - ((score - NEUTRAL_SCORE) * DECAY_RATE_BPS / 10 000)` (clamped to `[0, MAX_SCORE]`).
- MUST update `lastUpdatedAt`.
- MAY be called by any address; callers MAY be incentivised by a small keeper reward.
- MUST emit `DecayApplied(agent, score)`.

### Events

```solidity
event TaskCompleted(address indexed agent, bytes32 indexed escrowId, uint256 newScore);
event DisputeLost(address indexed agent, bytes32 indexed escrowId, uint256 newScore);
event ReputationStaked(address indexed agent, uint256 amount, uint256 newScore);
event ReputationUnstaked(address indexed agent, uint256 amount, uint256 newScore);
event DecayApplied(address indexed agent, uint256 newScore);
```

### Whitelisted Reporter Registry

Implementations MUST maintain a mapping of whitelisted reporter contracts. Only reporters in this set MAY call `recordTaskCompletion` and `recordDisputeLoss`. The owner or governance module MUST be able to add and remove reporters.

```solidity
event ReporterAdded(address indexed reporter);
event ReporterRemoved(address indexed reporter);
```

### [ERC-165](./eip-165.md)

Compliant contracts MUST implement the `IERC8203` interface and MUST return `true` from `supportsInterface` for `type(IERC8203).interfaceId`.

## Rationale

* **Decay toward neutral** — Absolute decay (decrease to zero) penalises inactive agents too harshly. Decay toward `NEUTRAL_SCORE` allows dormant agents to resume activity without a permanently crippled score.
* **Dispute asymmetry** — `DISPUTE_PENALTY (500) > TASK_REWARD (50)` by a factor of 10, matching economic literature on reputation catastrophe avoidance.
* **Stake multiplier cap** — The cap of 1 000 BPS (10 %) prevents wealthy agents from saturating the score via staking alone.
* **Reporter whitelist** — Keeping the update surface narrow prevents score inflation from counterfeit escrow contracts.
* **Read-only `getScore`** — Virtual decay in the view function removes the need for a global cron and keeps gas costs predictable for on-chain consumers.

## Backwards Compatibility

No existing ERC is modified. Deployments of other Opacus standards that persist a `kineticScore` field (e.g., ERC-8201 agent records) MAY migrate to querying ERC-8203 in place of storing the field locally.

## Security Considerations

* **Reporter whitelist poisoning** — Governance processes MUST apply timelocks before adding new reporters to prevent flash-governance attacks.
* **Stake reentrancy** — `unstakeReputation` transfers ETH and MUST be protected by a reentrancy guard.
* **Integer underflow** — `score - DISPUTE_PENALTY` MUST be floored at `0` with checked or saturating arithmetic.
* **Decay manipulation** — `applyDecay` is publicly callable; implementations MUST ensure this cannot be used to grief individual agents with excessive gas consumption (bound the epoch loop).
* **Sybil via stake** — The stake multiplier is capped; Sybil resistance ultimately relies on bonded registration at the registry layer (ERC-8201) rather than ERC-8203 alone.
